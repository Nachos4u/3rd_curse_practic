# Прайм Бургер — система управления рестораном

Настольное приложение для автоматизации работы бургерной: бронирование столов,
ведение заказов с контролем склада, смены официантов, оплата и формирование чеков,
а также аналитические отчёты. Разработано в рамках учебной практики **УП 02.01**.

| | |
|---|---|
| **Платформа** | .NET 10, Windows Forms (C#) |
| **База данных** | PostgreSQL (схема в 3НФ, функции, триггеры, представления) |
| **Архитектура** | многослойная: `Core` → `Data` → `Services` → `UI` |
| **Тесты** | xUnit (модульные и интеграционные) |

## Возможности

- **Регистрация и авторизация** с разграничением прав по ролям (Администратор, Официант, Кухня, Клиент). Пароли хранятся только в виде хеша PBKDF2-SHA256.
- **Бронирование столов** на дату и время с контролем вместимости (≤ 4 мест за столом), рабочего времени (9:00–23:00) и пересечений броней; отмена брони.
- **Заказы**: создание, добавление/удаление блюд с контролем остатка на «складе» кухни, оформление с фиксацией даты и времени, отмена (пока кухня не приступила к готовке).
- **Кухня**: очередь заказов, смена статусов «Готовится» → «Готов к выдаче».
- **Счета и чеки**: формирование счёта по готовым заказам стола, оплата, печать чека.
- **Смены официантов**: планирование администратором с прикреплением столов, открытие и закрытие смены официантом.
- **Меню и склад**: добавление блюд, пополнение склада, изменение цен, акции/скидки.
- **Схема зала**: графическое отображение столиков и их доступности на выбранный момент времени.
- **Статистика** — 6 отчётов: продажи блюд за два месяца с динамикой, брони столов, работа официантов, свободные столы, занятость столов, почасовая занятость.

## Архитектура и состав решения

```
RestaurantApp.slnx
├─ src/
│  ├─ Restaurant.Core      — доменные модели, перечисления, хеширование паролей
│  ├─ Restaurant.Data      — доступ к PostgreSQL (Npgsql + Dapper), репозитории
│  ├─ Restaurant.Services  — бизнес-логика и правила
│  └─ Restaurant.UI        — интерфейс пользователя (Windows Forms)
├─ tests/
│  └─ Restaurant.Tests     — модульные и интеграционные тесты (xUnit)
└─ db/
   ├─ 01_schema.sql        — таблицы, ключи, ограничения
   ├─ 02_views_functions.sql — представления, функции, триггеры, отчёты
   └─ 03_seed.sql          — демонстрационные данные
```

## Установка и запуск

### 1. Развёртывание базы данных

Создайте базу и примените скрипты (по умолчанию используется пользователь `postgres`):

```bash
psql -U postgres -c "CREATE DATABASE restaurant_db ENCODING 'UTF8';"
psql -U postgres -d restaurant_db -f db/01_schema.sql
psql -U postgres -d restaurant_db -f db/02_views_functions.sql
psql -U postgres -d restaurant_db -f db/03_seed.sql
```

### 2. Настройка подключения

По умолчанию приложение подключается к
`Host=localhost;Port=5432;Database=restaurant_db;Username=postgres`.
Строку подключения можно переопределить переменной окружения:

```bash
set RESTAURANT_DB=Host=localhost;Port=5432;Database=restaurant_db;Username=postgres;Password=ВАШ_ПАРОЛЬ
```

### 3. Сборка и запуск

```bash
dotnet build RestaurantApp.slnx
dotnet run --project src/Restaurant.UI
```

## Руководство пользователя по ролям

### Клиент
1. Войдите под учётной записью клиента или зарегистрируйтесь.
2. **Схема зала** — выберите дату и время, чтобы увидеть свободные (зелёные) и забронированные (красные) столы.
3. **Мои брони** — задайте дату, время, число гостей, отметьте столы и нажмите **«Создать бронь»**. Бронь можно отменить, выбрав её в списке.

### Официант
1. **Моя смена** — откройте смену в начале рабочего дня (смена должна быть заранее запланирована администратором) и закройте её в конце.
2. **Заказы** — нажмите **«Новый заказ»**, выберите стол, добавляйте блюда (контролируется остаток на складе) и нажмите **«Оформить заказ»**. Пока заказ в статусе «Составление», состав можно менять; отменить можно до начала готовки.
3. **Счета и оплата** — для готовых к выдаче заказов сформируйте счёт, затем выполните оплату; система сформирует чек.
4. **Брони** и **Схема зала** доступны для просмотра и создания броней гостям.

### Кухня
1. Откройте вкладку **Кухня** — отображается очередь оформленных заказов.
2. Выберите заказ и нажмите **«Взять в готовку»** (статус «Готовится»), затем **«Готов к выдаче»**.

### Администратор
1. **Меню и склад** — добавляйте блюда, пополняйте остатки, меняйте цены.
2. **Смены** — выберите дату и официанта, прикрепите столы и нажмите **«Запланировать смену»**.
3. **Пользователи** — создавайте учётные записи с нужной ролью, включайте/отключайте доступ.
4. **Отчёты** — формируйте 6 видов статистики с выбором периодов.
5. **Брони** и **Схема зала** — полный просмотр загрузки зала.

## Тестирование

```bash
dotnet test
```

Тесты делятся на:
- **модульные** — логика хеширования паролей, форматирование статусов и ФИО (выполняются без БД);
- **интеграционные** — авторизация, правила бронирования и отчёты на демонстрационной базе (пропускаются, если БД недоступна).
